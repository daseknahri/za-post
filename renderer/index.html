Settings

Configure automation behavior

⚡ Speed ▸
One fleet-wide baseline — each account can Inherit it or override on the Accounts tab. 🛡️ Safe = full human behavior (real typing, reading pauses, full anti-spam gaps): safest, for cold / warming / reserve accounts. ⚡ Fast = pastes + skips the reading pauses but KEEPS the full anti-spam gaps: the quickest still-safe pace, for warmed accounts. 🚀 Max = everything pasted, smallest gaps, aggressive: highest ban risk (all-warmed fleet only; post the first item manually). Quick Setup uses the same three.
🖼️ Content & variation ▸
Attach the post's image to its first comment too. Only applies to posts that have a comment.
{option1|option2} blocks resolve randomly per post — avoids duplicate-content flags.
Invisible per-group pixel tweak so each image has a unique hash — reduces duplicate-media detection.
Adds a random tracking parameter so each URL is unique and harder to flag as repetitive spam.
🌱 Accounts & warm-up ▸
New accounts browse and interact lightly before their first post to appear more human to Facebook.
How many successful runs an account is treated as "new" and warmed (only used when warm-up is on).
Max GROUP-posts per account per day (counts each group). 0 = no cap. For "1 post into all my groups daily" set this to the number of groups each account has (or leave 0 and use Daily mode).
Base hours a rate-limited account rests before retrying; doubles each repeat strike (capped at 48h).
Hold back this many healthy POSTERS each cycle as extra reserves (rotated). 0 = only your Standby accounts are reserves (all posters post). Standby accounts are ALWAYS reserves regardless of this number.
How many drops ONE reserve may cover in a single cycle. 1 (default) = one job per account/cycle (safest). Raise to 2–5 so a healthy reserve covers multiple dropped accounts when drops outnumber reserves.
How many accounts post simultaneously from your one shared IP (this is why you see N browsers open at once). The real number is min("Parallel accounts", this) — so to go above 3 you must raise both. 3 (default) ≈ a normal household's devices, the safe balance. Raise for more SPEED (up to 8), but more simultaneous Facebook sessions from one line is a stronger spam/correlation signal — dial back if blocks appear. Also limited by free RAM (~300–500 MB per browser). Proxied accounts ignore this.
Anti-ban ceiling on your whole fleet's posting rate from one shared IP (only when running WITHOUT proxies). Spaces every post so no two land closer than this many seconds across all accounts — e.g. 60 ≈ ~60 posts/hour max from your line. Default 15s (anti-burst hardening for one shared IP — barely slows a 3-browser run); 0 = off (per-account pacing only). Only ever slows; never changes which posts go where. Raise it if Facebook starts blocking; lower toward 0 for max speed.
Make each PROXIED account's browser report its proxy's timezone + language instead of this computer's — a mismatch Facebook can use to link accounts. Applies to every proxied account that has no per-account override (Accounts tab). Leave blank for no override.
Routes a lookup through each proxy to learn its region, then auto-fills each account's timezone/locale (and these fields if all proxies share one region). Proxies you add later are detected automatically.
🕒 Automation & schedule ▸
Your accounts start their daily cycle(s) at this time.
How many cycles each run delivers, then it's done for the day (the next run auto-starts at the time above). Each cycle every account posts its NEXT post — so 5 = each account posts 5 of its posts this run, spaced by the gap below. Never the same post twice; posts stay spaced for safety.
The gap between cycles within a run (jittered up to +30% so the cadence isn't a fixed spam tell). Lower = the run's cycles fire closer together. Min 0.5 = 30s (e.g. 0.5=30s, 1=1 min). ⚠️ Gaps under ~5 min raise ban risk on a shared IP — more total posts/hour from one line; the per-post anti-spam gaps still pace each post, but overall velocity goes up. Leave blank to use your speed preset's timing. (Also sets the between-cycle gap in Continuous mode.)
1 = classic (one group at a time). 2–4 = while a group is posting, the next group(s) pre-load in background tabs so each account finishes faster. Posting itself stays one-at-a-time and paced — this only overlaps the slow page-loading, so there's no extra spam/velocity risk. More tabs = faster but more RAM.
Each account posts the image+caption to all its groups first, then goes back and adds every post's comment. The time spent posting the other groups becomes the wait before commenting — so the per-post comment delay is absorbed for free, and every post lands before any comment work. Posts still fire one-at-a-time and paced; the comment can never double-post (the post is already counted). Combine with parallel tabs above. Held ("Spam potentiel") posts and any blocked comment route to a reserve/moderator exactly as normal.
Two-phase only. Instead of reloading the group and scrolling to find the post to read its link (slow, and Facebook often hides the link in the feed), read the link straight from Facebook's own publish response the moment the post is created — no reload, no scroll. Before commenting, the app still opens that link and re-checks the post is yours (caption + author), so a wrong link can never comment on the wrong post — it just falls back to the old feed search. Turn on and watch the log for "Captured the post's link from Facebook's publish response"; if you see it, it's working.
Registers a Windows scheduled task that launches the app and runs the daily post at the time above — even if the app is closed. Best for an unattended/RDP laptop.
If the app was closed mid-run (shutdown or crash), resume the campaign on next launch. Recommended ON for an unattended laptop.
Registers the app as a Windows login item so it starts when you sign in.

🛡️ Moderation & delivery rescue ▸
For groups you ADMIN: a designated moderator account approves posts FB holds in "Spam potentiel"/pending, so the post goes live and its comment can land. It approves only posts that match YOUR captions (and your accounts' FB display names) — never a stranger's. Turn on Test mode first to see what it would approve.
When a post stays stuck in "Spam potentiel" and the moderator can't approve it, a healthy reserve account (member of that group) re-posts the content so it goes live. Checks the feed first so it never duplicates, caps at one replacement per post.
How long to wait after a post is marked un-approvable before a reserve re-posts it (0–86400 s). Default 180 s.

🌐 Import from Chrome

Brings your already-logged-in Chrome profiles into the app with their device identity (the datr cookie), so Facebook keeps seeing the same account on the same device — no re-login, no "new device" checkpoint. Modern Chrome encrypts sessions so this must go through a tiny helper extension (localhost only — nothing leaves this PC).

    Generate & open the helper
  2. 2.
    Load it in Chrome (once)
    Open chrome://extensions → turn on Developer mode → Load unpacked → pick the folder from step 1. It applies to all your profiles.
  3. 3.
    Open each profile on Facebook
    Open a profile on facebook.com — the helper auto-sends that account here (or click its toolbar icon and “Send”). Accounts appear below. Tip: in the helper popup set each profile’s label (e.g. BB24) to name it, and optionally its login for an auto re-login safety net.
